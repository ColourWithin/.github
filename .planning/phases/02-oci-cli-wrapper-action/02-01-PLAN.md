Behavior:
- trim `command`
- fail if empty
- parse with `shell-quote.parse`
- require first accepted token to be exactly `oci`
- reject env-assignment prefixes such as `OCI_CONFIG=... oci ...`
- reject every non-string token returned by `shell-quote`
- reject separators/operators/control tokens including newline command separators, `;`, `&&`, `||`, redirects, comments, substitutions, and glob/non-string artifacts
- if `query` input is non-empty and parsed args already include `--query`, throw an error containing `query input cannot be used when command already includes --query`
- if `query` input is non-empty and command has no `--query`, append `--query` and the exact query value as a separate argv token
- resolve relative `working-directory` against `GITHUB_WORKSPACE`
- fail if `working-directory` does not exist or is not a directory

Tests must cover valid simple commands, spaces, quoted JMESPath, inline JSON for `--from-json`, nested JSON strings, query append, query conflict, non-oci command, empty command, env assignment prefix, shell separators, redirects, comments, substitutions, non-string shell-quote tokens, missing working directory, file-as-working-directory, and relative workspace resolution.
