  <action>
Create `docs/actions/oci-ipt-smoke.md` with these sections:
- `# OCI IPT Smoke Test`
- `## Purpose` explaining that the real smoke lives in `colour-within-ops` because the IPT trusts that repo, and that the workflow proves auth only.
- `## Workflow location` naming `colour-within-ops/.github/workflows/oci-ipt-smoke.yml`.
- `## Required workflow properties` listing `workflow_dispatch`, `environment: production`, and:

```yaml
permissions:
  id-token: write
  contents: read
```

- `## Workflow skeleton` containing a complete YAML skeleton with:
  - `on.workflow_dispatch.inputs.candidate_sha.required: true`
  - token exchange step:
    `uses: ColourWithin/.github/actions/oci-token-exchange@${{ inputs.candidate_sha }}`
    with `audience: https://github.com/ColourWithin/colour-within-ops`
  - CLI smoke step:
    `uses: ColourWithin/.github/actions/run-oci-cli-command@${{ inputs.candidate_sha }}`
    with `command: oci os ns get`
  - verification step that parses the smoke output as JSON using Python or `jq`, verifies `steps.smoke.outputs.exit-code` equals `0`, verifies `steps.smoke.outputs.oci-cli-version` is at least `3.81.1`, and writes a Markdown summary to `$GITHUB_STEP_SUMMARY`.
- `## Fallback read command` documenting `oci iam compartment list --compartment-id <secret-supplied-ocid>` using `secrets.OCI_SMOKE_COMPARTMENT_OCID` only if the ops team needs a more permission-scoped proof.
- `## Evidence to record before v1.0.0 tag` with fields: candidate SHA, workflow run URL, workflow path, command, exit code, `oci-cli-version`, JSON parse result, operator/date.
- `## Non-goals` explicitly listing no deploy, no mutation, no IPT trust expansion to `.github`, and no cross-repo status bridge in v1.0.
  </action>
